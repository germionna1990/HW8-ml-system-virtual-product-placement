# Домашнее задание HW8 — Мониторинг ML-систем

Репозиторий содержит выполненное домашнее задание по теме мониторинга ML-систем, контроля качества данных и проектирования ML-архитектуры.

## Что выполнено

### 1. Определение бизнес- и технических метрик ML-системы

Для ML-сервиса было сформировано дерево метрик.

Рассмотрены:

**Бизнес-метрики:**
- CTR рекомендаций
- Conversion Rate
- ARPU
- удержание пользователей

**Технические метрики:**
- latency
- error rate
- availability
- throughput

Также определены целевые значения для мониторинга сервиса:
- p95 latency < 1 сек
- Error Rate < 1%
- Availability > 99%

---

### 2. Настройка мониторинга ML-сервиса

Реализован демонстрационный сервис мониторинга.

Что сделано:
- создан FastAPI-сервис
- endpoint `/predict` для имитации ML-инференса
- endpoint `/metrics` для Prometheus
- настройка Prometheus
- настройка Grafana
- запуск инфраструктуры через Docker Compose

Используемый стек:
- FastAPI
- Prometheus
- Grafana
- Docker Compose

Файлы:
- `monitoring/app.py`
- `monitoring/prometheus.yml`
- `monitoring/docker-compose.yml`

---

### 3. Обнаружение деградации модели и drift данных

Для демонстрации data drift использован инструмент Evidently.

Что сделано:
- создан эталонный набор данных (`reference_data`)
- создан измененный набор (`current_data`)
- искусственно изменены распределения признаков:
  - возраст
  - зарплата
  - score
- построен отчёт Data Drift

Используемый стек:
- Python
- pandas
- numpy
- Evidently

---

### 4. Контроль качества данных

Рассмотрен сценарий контроля качества данных.

Что сделано:
- попытка запуска DQOps в Google Colab
- из-за ограничений лицензирования полноценный запуск не завершился
- для демонстрации сценария использована SQL-проверка качества данных

Реализован пример:
- создание таблицы `customers`
- заполнение тестовыми данными
- изменение структуры таблицы как пример инцидента качества данных

Используемый стек:
- SQL
- DQOps (ознакомительно)

---

### 5. Проектирование ML-системы Virtual Product Placement

Спроектирована архитектура ML-системы для виртуального product placement.

Что реализовано:
- загрузка тестового видео
- детекция человека через YOLO
- потоковая обработка кадров
- Redis broker
- FastStream publisher/subscriber pipeline
- генерация обработанного видео
- архитектурная диаграмма системы

Выбрана **Kappa Architecture**, так как обработка выполняется как единый поток событий в реальном времени.

Используемый стек:
- YOLO (Ultralytics)
- OpenCV
- Redis
- FastStream
- Graphviz / Diagrams
